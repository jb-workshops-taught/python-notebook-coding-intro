How does a computer make decisions?

A computer does different things depending on circumstances. For instance, if your online shopping cart is empty, it might show "Empty cart" but if it has items in it, it shows those items. How does it decide?

If Statements and Troubleshooting

What if we want to make the computer say something different, depending on what the user at the keyboard typed in? How do we do that? Try this example. What does it do?

time = input("Is it morning or night? ") if(time == "morning"): print("Good morning!") else: print("Have a great night.")

Notice how we use a single = when we're assigning a variable's value, but we use a double == (two equal signs in a row) when we're comparing two values to ask the question, "Are these equal? Are they the same?"

The comparison gives us a true/false or yes/no answer. That's called a boolean value. Here are some variables that hold boolean values. What does this code print? Why?

a = 5 b = 5 same = (a == b) print("Are they the same? ", same) greater = (a > b) print("Is a greater than b? ", greater) lesser = (a < 9) print("Is a less than 9? ", lesser)

Here's a deeper explanation of comparisons. There are several kinds of comparisons that a computer can do. It can compare whether numbers are the same, less than, or greater than. It can also compare as "less than or equal to" and as "greater than or equal to". Numbers are the easiest to compare, but letters can be compared as well, which lets us sort things alphabetically.

You'll use those comparison operators a lot. Operator means the punctuation marks used to make a comparison, for example, < > == and !=. Eventually you will have these memorized, from practicing with them and using them so much.

Now let's look at a longer version of the first example (below). When the user types in "morning", then the first block of code indented inside the if statement will run. This is because the (time == "morning") condition is true. When the if condition is true, the first indented block runs; otherwise, the else indented block runs.

The final line is not indented, so it's not part of the if-else statement. No matter what the if statement does, the last line prints anyway.

Try running it. The first time, put in morning, and watch what it prints. Run it again and put in night, and watch what it prints. What is the same both times? What is different?

You can nest if statements as well. Watch the indentation! Run this a few times and type in different answers. Try to understand how it works. (Note: elif is a contraction from "else if".)

time = input("What time of day is it? Answer day or night: ") season = input("What season is it? Answer winter, summer, or spring: ") print("") if (season == "winter"): if(time == "day"): print("Winter in the daytime can be very bright with snow.") else: print("Winter at night is beautiful with moonlight on the snow.") elif (season == "summer"): if(time == "day"): print("Summer days are for play!") else: print("Summer nights are warm but not bright.") else: if(time == "day"): print("Spring flowers are so colorful in the daytime rain.") else: print("Spring nights are chilly.") print("See you later!")

Troubleshooting Strategies

As you work with code, you'll encounter mistakes, typos, and misunderstandings. Some of these will put errors up in your face in Jupyter, and others will result in incorrect print output but otherwise no indicator of what went wrong. How do you find the problem?

Use print statements to inspect variables, to find out if they really hold the value you think they do.

Break the problem down into smaller steps. Check your logic on each part before bringing the parts together.

Break your code apart into separate boxes, no more than a few lines in each one, and run the parts separately (but in the correct order). Print the values you're curious about at the end of each box. This helps you narrow down where the problem is.

Write your ideas out on paper, step by step, in English. Make sure they are coherent and make sense before you try to write the code to match.

Come up with test values to put in, like we did with candy prices. Make sure the math checks out on paper. Make sure you know how it behaves for weird inputs like zero, one, or negative one.

(How many candies can you get for zero cents each? Try it.) Make sure you understand any issues with decimal rounding as well (hint, hint).

If Statements: Concepts Summary

  1. An if statement makes a decision based on a true-false boolean condition.
  2. If the condition is true, the first indented branch runs.
  3. If the condition is false, the (optional) else branch runs.
  4. Indenting code is how you indicate which part belongs to the if statement.
  5. The best way to troubleshoot problems is to work carefully in a small section at a time.
